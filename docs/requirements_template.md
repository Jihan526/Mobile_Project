## 1. 프로젝트 개요

### 1.1 프로젝트 목표
> 일상 속에서 친구, 가족 등 에게 가벼운 장난이나 칭찬, 감사 등을 전하고 싶을 때 단순한 메신저 보다 상장이라는 재미있는 방법으로 일상생활에서 소소한 이벤트를 만들어줄 수 있도록 한다.

### 1.2 대상 사용자
> 재미있는 방법으로 지인에게 마음을 전하고 싶은 남녀노소 누구나

### 1.3 핵심 가치
> 텍스트 몇 글자만으로 Gemini API가 상장 본문을 자동 생성하고, 커스텀 사진을 통해 사용자가 원하는 상장 이미지를 제작하고 공유할 수 있다.

## 2. 기능 요구사항 (Functional Requirements)

> 시스템이 *무엇을 하는가*를 정의한다. 최소 5개 이상 작성
> 문장 패턴: "사용자는 ~할 수 있다" / "시스템은 ~한다"

### FR-01: [AI 상장 문구 자동 생성]

- **설명**: 사용자가 상장 수여 사유를 짧게 입력하면,자연스러운 문장으로 본문 텍스트를 자동 생성할 수 있다.
- **수용 조건**:
  - [ ] 텍스트 입력창, 생성 요청 버튼이 제공되어야 한다.
  - [ ] 시스템은 Gemini API를 호출하여 입력된 사유를 상장 문구 형태로 변화해야한다.
  - [ ] 상장 수여 사유를 입력받아 변환하는 동안 사용자에게 로딩 상태를 시각적으로 안내해야 한다.

### FR-02: [상장 템플릿 선택]

- **설명**: 사용자는 시스템이 제공하는 다양한 상장 디자인 템플릿 중 하나를 선택하여 배경으로 적용할 수 있다.
- **수용 조건**:
  - [ ] 3개 이상의 기본 상장 배경 이미지가 목록 형태로 제공되어야 한다.
  - [ ] 탬플릿 선택 시 메인 화면의 상장 배경 이미지가 즉각적으로 변경되어야 한다.

### FR-03: [커스텀 배경 이미지 촬영/로드 및 편집 기능]

- **설명**: 사용자는 스마트폰 카메라로 직접 사진을 촬영하거나 갤러리에 저장된 사진을 불러와 상장 배경이나 장식 요소로 등록할 수 있다.
- **수용 조건**:
  - [ ] Camera Intent를 호출하여 촬영한 이미지를 결과값으로 받아 뷰에 띄울 수 있어야 한다.
  - [ ] Gallery Intent를 호출하여 저장된 이미지 파일을 선택하고 화면에 불러올 수 있어야 한다.
  - [ ] 불러온 이미지를 시스템 Crop Intent로 전달하여 사용자가 원하는 비율과 크기로 자를 수 있는 등 편집 환경을 제공해야 한다.
  - [ ] 사용자가 편집을 완료하면, 최종 편집이 완료된 이미지만 상장 영역에 매끄럽게 배치되어야 한다.

### FR-04: [커스텀 상장 갤러리 저장]

- **설명**: 시스템은 사용자가 제작한 커스텀 상장을 기기의 사진 캘러리에 이미지 파일로 저장한다.
- **수용 조건**:
  - [ ] 이미지 저장 버튼 클릭 시 Android Canvas API를 활용하여 지정된 상장 뷰 영역을 비트맵으로 캡쳐해야 한다.
  - [ ] 캡처된 이미지가 갤러리에 저장되고, 저장이 완료되면 토스트 메시지가 출력되어야 한다.

### FR-05: [완성된 상장 SNS 공유]

- **설명**: 사용자는 저장된 상장 이미지를 안드로이드 공유 기능을 통해 다른 메신저나 SNS 앱으로 전송할 수 있다.
- **수용 조건**:
  - [ ] 공유 버튼 클릭 시 Android 기본 Share Intent가 호출되어야 한다.
  - [ ] 캡쳐된 이미지의 URI 데이터가 Intent에 정상적으로 담겨 카카오톡, 인스타그램 등의 공유 대상 앱 목록이 표시되어야 한다.

### FR-06: [상장 보관함 제공]

- **설명**: 시스템은 사용자가 지금까지 제작했던 상장들을 모아볼 수 있는 보관함 기능을 제공하고, 사용자는 보관함에서 원하는 상장을 선택해 언제든지 갤러리에 저장하거나 SNS에 공유할 수 있다.
- **수용 조건**:
  - [ ] 완성된 상장 이미지는 기기 내부 저장소에 자동 저장되어야 한다.
  - [ ] 보관함 화면 진입 시, 저장된 상장 목록이 리스트나 그리드 형태로 출력되어야 한다.
  - [ ] 보관함 내의 특정 상장을 클릭했을 때, FR-04와 FR-05의 기능이 동일하게 작동해야 한다.

### FR-07: [이용 등급별 상장 제작 횟수 제한]

- **설명**: 시스템은 사용자의 등급(비회원, 회원, 프리미엄)에 따라 제작 가능한 상장의 최대 개수를 제한하고, 한도 초과 시 업그레이드 안내 창을 표시한다.
- **수용 조건**:
  - [ ] 비회원 상태일 경우, 누적 상장 제작 개수를 최대 2개로 제한한다.
  - [ ] 회원가입을 완료한 일반 회원 상태일 경우, 누적 상장 제작 개수를 최대 6개로 제한한다.
  - [ ] 프리미엄 등급을 구독한 유저는 제한 없이 무제한으로 상장을 제작할 수 있다.
  - [ ] 각 등급별 제한 횟수를 초과하여 제작을 시도할 경우, 비회원일 경우는 회원가입 페이지로 이동하고, 회원일 경우 결제 페이지로 이동을 유도하는 안내 알림창을 출력해야 한다.

## 3. 비기능 요구사항 (Non-Functional Requirements)

### 3.1 성능 (Performance)

- Gemini API를 통한 텍스트 생성 요청 시 평균 3초 이내에 결과를 화면에 출력해야 하며, 네트워크 지연 시 적절한 타임아웃 처리를 수행해야 한다.
- Canvas API를 통한 이미지 캡처 및 내부 저장소 작업이 메인 UI 스레드를 차단하지 않도록 백그라운드에서 비동기로 처리해야 한다.
- FR-6 구현 시 많은 수의 상장이 누적되더라도 화면 스크롤 시 끊김 현상 없이 부드러운 렌더링 성능을 유지해야 한다.

### 3.2 보안 (Security)

- 외부 API 호출에 필요한 구글 Gemini API Key는 소스 코드에 하드코딩하지 않고, 외부 오픈소스 저장소(GitHub 등)에 유출되지 않도록 해야 한다.
- 사용자의 현재 이용 등급 및 잔여 제작 횟수 정보(FR-07)의 값들은 사용자가 임의로 데이터를 조작하거나 위변조 할 수 없도록 안전하게 관리해야 한다.

### 3.3 사용성 (Usability)

- 비회원 및 일반 회원이 제작 한도에 도달했을 때 출력되는 유도 팝업창은 강압적이지 않고, 등급 상향 시 얻을 수 있는 혜택을 시각적으로 친근하게 전달해야 한다.
- 사용자가 현재 본인의 이용 등급과 남은 상장 제작 가능 개수를 앱 마이페이지 또는 메인 화면에서 한눈에 파악할 수 있도록 UI/UX를 설계한다.

### 3.4 호환성 (Compatibility)

- 최소 지원 SDK는 Android API 24 이상으로 설정하여 대다수의 최신 기기 파편화 환경에서 핵심 기능이 일관되게 동작하도록 해야한다.
- 가상 에뮬레이터 환경뿐만 아닌, 다양한 카메라 렌즈 비율 및 갤러리 URI 접근 권한을 가진 실제 스마트폰 기종들에서도 Camera/Gallery Intent가 충돌 없이 유연하게 연동되도록 예외 코드를 작성해야한다.

### 3.5 기타

- 앱이 강제 종료되거나 스마트폰 기기를 재부팅하더라도 기존 보관함에 저장된 상장 이미지 파일과 유저의 현재 등급/제작 제한 개수 데이터는 누락이나 초기화 없이 완벽하게 복구 및 유지되어야 한다.
